# Plan: US-004 Backend — Auto-fill Schedule

## Context

US-001 (Team Members), US-002 (Schedule Requirements), and US-003 (Schedule Entries) are complete. US-004 introduces automatic team member assignment: given a week with existing schedule entries (created via `set-schedule-entries`), the system reads each entry's structure, assigns team members fairly, and reports gaps.

**Key change**: `set-schedule-entries` currently handles both structure setup AND team member assignment. We split responsibilities: `set-schedule-entries` only handles structure + schedule requirements (no team members), and the new `auto-fill-schedule` use case handles team member assignment exclusively.

**PRD requirements (AC-004-A through AC-004-F)**:
- Auto-fill the schedule for a selected week
- Only assign team members whose profession AND specialty match
- Distribute assignments fairly (avoid scheduling same person every day)
- Report when requirements can't be fully met (which profession/specialty is short)
- Persist generated schedule entries in SQLite

---

## Design Decisions

### 1. Auto-fill updates existing entries, does not delete/recreate
Schedule entries are created via `set-schedule-entries` with their structure already defined. Auto-fill reads the structure from existing entries and assigns team members to them. No need to pull schedule requirements — the structure field contains all needed info.

### 2. Fair distribution algorithm
Process days in date order. For each day, fill specialty slots first (most constrained — sorted by fewest eligible candidates). Then fill remaining unreserved profession slots. Always pick from team members with the fewest total assignments across the week (ascending sort by assignment count, with deterministic tiebreak by ID).

### 3. Remove `teamMemberIds` from `set-schedule-entries`
Remove the field from payload, Zod schema, and all team member validation/resolution logic. On create: pass empty team members WatchedList. On update: don't touch existing team member assignments (make `teamMembers` optional in `UpdateScheduleEntryInput`).

### 4. API: `POST /schedule-entries/auto-fill`
Accepts `{ weekStartDate: string }` (must be a Monday within allowed set range). Returns updated entries + gap report.

---

## Auto-fill Algorithm (Pseudocode)

```
1. Validate weekStartDate is a Monday, compute Mon-Sun dates
2. Validate all 7 dates within getAllowedSetRange()
3. Fetch existing schedule entries for the date range
4. Load all team members
5. Initialize assignmentCounts: Map<teamMemberId, 0>
6. For each date (sorted) that has an existing entry:
   Read structure from entry.props.structure.value
   For each profession in structure:
     PHASE 1 — Fill specialty slots (most constrained first):
       Sort specialtyRequirements by fewest eligible team members
       For each specialty: pick from eligible (matching profession+specialty, not already assigned today)
         sorted by assignmentCounts ascending → take requiredCount
         Record gap if insufficient
     PHASE 2 — Fill remaining unreserved profession slots:
       remainingToFill = professionRequiredCount - alreadyAssignedForProfession
       Pick from eligible (matching profession, not already assigned today)
         sorted by assignmentCounts ascending → take remainingToFill
         Record gap if insufficient
7. Update each entry with team member assignments via WatchedList
8. Return { items, gapReport }
```

## Gap Report Response Shape

```ts
{
  hasGaps: boolean
  days: {
    date: string
    professionGaps: {
      profession: Profession
      requiredCount: number
      assignedCount: number
      deficit: number
      specialtyGaps: {
        specialty: Specialty
        requiredCount: number
        assignedCount: number
        deficit: number
      }[]
    }[]
  }[]
}
```

## API Contract

| Method | Path | Body | Success | Errors |
|--------|------|------|---------|--------|
| POST | `/schedule-entries/auto-fill` | `{ weekStartDate: "YYYY-MM-DD" }` | 200 `{ entries, gapReport }` | 400 (not Monday, outside range) |

The modified `set-schedule-entries`:

| Method | Path | Body | Success | Errors |
|--------|------|------|---------|--------|
| PUT | `/schedule-entries` | `{ dates, structure, scheduleRequirementIds }` (no `teamMemberIds`) | 200 | 400 |

## Files Created (2 new)

### `backend/src/domain/use-cases/schedule-entry/auto-fill-schedule.ts`
The main use case. Constructor takes `ScheduleEntryRepository`, `TeamMemberRepository`.

Payload: `{ weekStartDate: string }`
Response: `{ items: ScheduleEntry[], gapReport: AutoFillGapReport }`

Steps:
1. Validate weekStartDate is Monday, compute week dates, validate in set range
2. Fetch existing entries for the week via `findByDateRange()`
3. Load all team members
4. Run fair-distribution algorithm using each entry's structure
5. Update each entry with assigned team members via WatchedList
6. Return items + gap report

### `backend/src/infra/http/routes/auto-fill-schedule.ts`
Route handler for `POST /schedule-entries/auto-fill`.
Zod schema: `{ weekStartDate: z.string().regex(/^\d{4}-\d{2}-\d{2}$/) }`
Instantiates 2 Prisma repositories + use case. Returns 200 with presented entries + gap report.
