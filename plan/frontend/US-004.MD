# Plan: US-004 Frontend — Auto-fill Schedule

## Context

The backend US-004 is already implemented and introduced two breaking changes that affect the frontend:

1. **`PUT /schedule-entries` no longer accepts `teamMemberIds`** — The payload is now `{ dates, structure, scheduleRequirementIds }` only. The frontend still sends `teamMemberIds` and has a 2-stage dialog (stage 2 = team member selection), which is now broken.

2. **New endpoint `POST /schedule-entries/auto-fill`** — Accepts `{ weekStartDate: "YYYY-MM-DD" }` (must be a Monday), returns `{ entries: ScheduleEntry[], gapReport: AutoFillGapReport }`. This is how team members are now assigned to schedules.

This plan fixes the broken SetScheduleDialog and adds the new Auto-fill UI.

- **React Components** (`.claude/skills/writing-code/coding-patterns/react-components.md`): Named exports, destructured props, `handle` prefix for event handlers, function declarations (not arrow functions) for components and handlers, object state with typed interface, `useDialogs` hook for modal lifecycle, Tailwind v4 with global theme tokens, lucide-react icons, utility functions outside component when pure

```ts
export async function autoFillSchedule(
  data: { weekStartDate: string },
): Promise<AutoFillResponse> {
  const response = await client.post<AutoFillResponse>(
    API_ROUTES.scheduleEntries.autoFill,
    data,
  )
  return response.data
}
```

## Phase 3: Auto-fill Dialog

### New file structure (follows page-structure.md — subfolder for medium/big component)

```
frontend/src/pages/Schedule/components/
  AutoFillDialog/
    AutoFillDialog.tsx      — Main dialog (week select → loading → result)
    WeekSelector.tsx        — Two selectable week cards (current/next)
    GapReportDisplay.tsx    — Renders gap report after auto-fill
```

### 3.1 — `WeekSelector.tsx`

- Props: `{ selectedWeek: string | null, onSelectWeek: (mondayISO: string) => void }` (destructured per react-components.md)
- Pure presentational component — utility functions outside the component
- Computes current and next week Monday dates using `getMondayOfWeek`, `formatDateToISO` from `schedule-dates.ts`
- Uses `getSundayOfWeek` + `formatDateDisplay` for date range labels
- Renders two clickable cards (radio-style single-select):
  - Selected: `ring-2 ring-primary-500 bg-primary-50` (using theme tokens per react-components.md)
  - Unselected: `border border-border/40 bg-surface hover:bg-neutral-50`
  - Each shows: "This Week" / "Next Week" label + date range (e.g., "Mon, Apr 6 — Sun, Apr 12")
- Handler: `handleSelectWeek` with `handle` prefix

### 3.2 — `GapReportDisplay.tsx`

- Props: `{ gapReport: AutoFillGapReport }` (destructured)
- Pure presentational — utility functions (`formatDateDisplay`, `PROFESSION_LABELS`, `formatSpecialtyLabel`) from entities/schedule-dates
- If `gapReport.hasGaps` is false: success banner (green themed)
- If `gapReport.hasGaps` is true: warning banner + per-day breakdown:
  - Each `DayGap`: date header via `formatDateDisplay`
  - Each `ProfessionGap`: profession label, `assignedCount/requiredCount`, deficit `Badge variant="error"`
  - Each `SpecialtyGap`: indented under profession, specialty label, counts, deficit
- Visual pattern mirrors existing `FulfillmentSection` / `ProfessionFulfillmentRow` from ScheduleOverview

### 3.3 — `AutoFillDialog.tsx`

- Props: `{ onClose: () => void, onSuccess: () => void }` (destructured per react-components.md)
- **Object state with typed interface** (per react-components.md):

```ts
type AutoFillDialogStage = 'select' | 'loading' | 'result'

interface AutoFillDialogState {
  stage: AutoFillDialogStage
  selectedWeek: string | null
  result: AutoFillResponse | null
  error: string | null
}
```

- Uses `useState<AutoFillDialogState>` with initial state
- Modal overlay pattern matching `SetScheduleDialog` (fixed inset-0, card-raised, `max-w-xl`)
- Icon: `Sparkles` from lucide-react (per react-components.md — use lucide-react for icons)
- No code comments (per general-coding-practices.md)

**Stage `select`:**
- Header: "Auto-fill Schedule" + "Automatically assign team members for a week"
- `WeekSelector` component
- Error banner (conditional, same pattern as SetScheduleDialog)
- Footer: Cancel (secondary) + "Auto-fill" (primary, disabled if `state.selectedWeek === null`)
- `handleAutoFill`: sets stage to `loading`, calls `autoFillSchedule({ weekStartDate: state.selectedWeek })`, on success sets stage to `result` with result data, on error sets stage back to `select` with error message

**Stage `loading`:**
- Centered `Loader2` spinner with "Generating schedule..." text
- No footer buttons

**Stage `result`:**
- Summary card: `"${state.result.entries.length} schedule entries updated"` (like ScheduleOverview header)
- Empty entries case: "No schedule entries found for this week. Use Set Schedule first."
- `GapReportDisplay` with `state.result.gapReport`
- Footer: "Close" button → `handleClose` calls `onSuccess()` then `onClose()`

**Error handling:**
- Parse 400 errors similar to `SetScheduleDialog.parseValidationError`
- Validation error messages for auto-fill specific codes (e.g., `NOT_MONDAY`, `DATES_OUTSIDE_ALLOWED_RANGE`)
