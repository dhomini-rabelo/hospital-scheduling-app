# Plan: US-001 — Manage Team Members (Frontend)

## Context

The backend for US-001 is fully implemented with CRUD endpoints at `/team-members` (port 5000). The frontend is a greenfield React 19 + TypeScript + Vite + Tailwind v4 app with only a design system (`global.css`) and a hello-world `App.tsx`. No routing, API client, components, or pages exist yet.

This plan implements the Team Members page (US-001) and the minimum layout/navigation needed (partial US-007).

## Dependencies to Install

In `frontend/`:
- `axios` — HTTP client
- `react-router` — routing (v7, imports from `react-router`)
- `@tanstack/react-query` — server state caching
- `react-hook-form` — form management
- `@hookform/resolvers` — zod resolver for react-hook-form
- `zod` — schema validation
- `lucide-react` — icons (per coding patterns)

## Implementation Phases

### Phase 0: Project Setup

**0.1** Add `vite-tsconfig-paths` plugin to `frontend/vite.config.ts` (already in root package.json).

**0.2** Install deps in `frontend/`:
```bash
npm install axios react-router @tanstack/react-query react-hook-form @hookform/resolvers zod lucide-react
```

### Phase 1: Core Infrastructure

**1.1** Create `frontend/src/core/api-client.ts`
- Axios instance with `baseURL: 'http://localhost:5000'`
- Default JSON headers

**1.2** Create `frontend/src/server/routes.ts` — API_ROUTES const
```ts
export const API_ROUTES = {
  teamMembers: {
    list: '/team-members',
    create: '/team-members',
    update: (id: string) => `/team-members/${id}`,
    delete: (id: string) => `/team-members/${id}`,
  },
} as const
```

**1.3** Create `frontend/src/hooks/useAPIRequest.ts`
- Wraps `@tanstack/react-query` `useQuery` (pattern from `plan/docs/hooks.md`)
- Uses axios `client` from `core/api-client.ts`
- Returns `{ actions: { refetch }, state: { data, isLoading, isError, error } }`

**1.4** Create `frontend/src/hooks/useDialogs.ts`
- Simple useState-based hook (per coding patterns)
- `useDialogs<T extends string>()` → `{ currentActiveDialog, activateDialog, disableDialog }`

### Phase 2: Domain Types

**2.1** Create `frontend/src/server/types/entities.ts`
- Enums: `Profession`, `DoctorSpecialty`, `NurseSpecialty`, `TechnicianSpecialty`, `SupportStaffSpecialty`
- `PROFESSION_SPECIALTIES` map (mirrors `backend/src/domain/entities/team-member.ts:44-49`)
- `PROFESSION_LABELS` display map
- `formatSpecialtyLabel()` helper (e.g. `GENERAL_WARD` → `General Ward`)
- `TeamMember` interface: `{ id, name, profession, specialty, createdAt, updatedAt }`

### Phase 3: API Layer

**3.1** Create `frontend/src/server/api/team-members.ts`
- `fetchTeamMembers(profession?)` → GET with optional query param
- `createTeamMembers(items[])` → POST with `{ items }` body
- `updateTeamMember(id, data)` → PUT
- `deleteTeamMember(id)` → DELETE
- All use `client` from `core/api-client.ts` and `API_ROUTES` from `server/routes.ts`

### Phase 4: Layout (partial US-007)

**4.1** Create `frontend/src/layout/components/Sidebar.tsx`
- Fixed left sidebar (`w-60`), dark background (`bg-neutral-900`)
- 3 nav links using `NavLink` from react-router: Schedule `/`, Schedule Requirements `/schedule-requirements`, Team Members `/team-members`
- Active link highlighted via NavLink's `isActive`
- Icons: `Calendar`, `ClipboardList`, `Users` from lucide-react

**4.2** Create `frontend/src/layout/components/layouts/AppLayout.tsx`
- Flex row: `<Sidebar />` + `<Outlet />` in scrollable content area
- `min-h-screen`, `bg-background`

**4.3** Create `frontend/src/layout/router/routes.ts`
```ts
export const ROUTES = {
  schedule: { route: '/', layoutRoute: '/' },
  scheduleRequirements: { route: '/schedule-requirements', layoutRoute: 'schedule-requirements' },
  teamMembers: { route: '/team-members', layoutRoute: 'team-members' },
} as const
```

**4.4** Create `frontend/src/layout/router/Router.tsx`
- Uses `BrowserRouter`, `Routes`, `Route` from `react-router`
- Root layout route with `AppLayout`
- Child routes using `ROUTES` layoutRoute values
- Pattern follows `plan/docs/router.md`

**4.5** Modify `frontend/src/core/App.tsx` — render `<AppRouter />`

**4.6** Modify `frontend/src/core/main.tsx` — wrap with `QueryClientProvider`

### Phase 5: Shared UI Components

All in `frontend/src/components/ui/`. Follow coding patterns: named exports, destructured props, Tailwind design tokens, `handle` prefix for handlers.

| File | Purpose |
|------|---------|
| `Button.tsx` | Variants: primary, secondary, danger, ghost. Props: `variant`, `size`, `isLoading`, `disabled`, `children`, `onClick`, `type` |
| `Input.tsx` | Text input with label, error message. Registers via react-hook-form `register` prop or standalone |
| `Select.tsx` | Native select with label, `options: {value, label}[]`, error message. Same register pattern |
| `Badge.tsx` | Colored pill using `.badge` class. `variant` prop for color |
| `ConfirmDialog.tsx` | Modal overlay. Props: `title`, `message`, `onConfirm`, `onCancel`, `isLoading` |

**6.2** Create `frontend/src/pages/ScheduleRequirements/page.tsx` — renders heading "Schedule Requirements" (placeholder)

### Phase 7: Team Members Page

Following `plan/docs/page-structure.md` structure at `frontend/src/pages/TeamMembers/`.

**7.1** `components/TeamMemberForm/TeamMemberFormRow.tsx`
- Single row within batch form: Name `<Input>`, Profession `<Select>`, Specialty `<Select>`
- Uses `useFormContext` from react-hook-form (FormProvider wraps parent) — avoids prop drilling form state
- Specialty options filtered by watched profession value via `PROFESSION_SPECIALTIES`
- Remove row button (X icon)

**7.2** `components/TeamMemberForm/TeamMemberForm.tsx`
- Batch creation form (AC-001-A, AC-001-H)
- Uses `useForm` + `FormProvider` with zod schema via `@hookform/resolvers/zod`
- Zod schema: `{ items: z.array(z.object({ name, profession, specialty })).min(1) }`
- `useFieldArray` for dynamic rows (add/remove)
- Submit calls `createTeamMembers` API, on success calls `onSuccess` callback
- Props: `onSuccess: () => void`

**7.3** `components/TeamMemberList/TeamMemberRow.tsx`
- Displays single team member: name, profession badge, specialty badge
- Edit and Delete icon buttons
- Props: `teamMember: TeamMember`, `onEdit`, `onDelete`

**7.4** `components/TeamMemberList/TeamMemberList.tsx`
- Renders list of `TeamMemberRow` components
- Empty state message when no members
- Props: `teamMembers: TeamMember[]`, `onEdit`, `onDelete`

**7.5** `components/EditTeamMemberDialog.tsx`
- Modal with form pre-filled from selected member
- Uses `useForm` with zod schema (name, profession, specialty — all optional for partial update)
- Specialty options filtered by profession (same pattern as creation form)
- Calls `updateTeamMember` API on submit
- Props: `teamMember: TeamMember`, `onClose`, `onSuccess`

**7.6** `page.tsx` — TeamMembersPage (page entry point)
- Uses `useAPIRequest` to fetch team members (react-query caching)
- Profession filter state → passed as `params` to `useAPIRequest`
- Uses `useDialogs<'edit' | 'delete'>()` for dialog management (simple useState, all in one file)
- Tracks `selectedMember: TeamMember | null` for edit/delete target
- Renders: page heading, profession filter `<Select>`, `<TeamMemberForm>`, `<TeamMemberList>`, `<EditTeamMemberDialog>` (conditional), `<ConfirmDialog>` (conditional for delete)
- After any mutation success: calls `refetch` from `useAPIRequest`
