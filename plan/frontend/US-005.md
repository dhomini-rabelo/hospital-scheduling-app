# Plan: US-005 Frontend — Swap Staff and Auto-fill Day Gaps

## Context

US-001 through US-004 are fully implemented (backend + frontend). US-005 backend is complete with 3 API endpoints: list swap candidates, swap team member, and auto-fill day gaps. The frontend currently has no swap or per-day auto-fill UI. This plan adds the ability to swap individual team members from the daily schedule and auto-fill gaps for a single day.

**Key constraint**: The `GET /schedule-overview` response does NOT include the schedule entry ID (only junction map IDs in `entries[].id`). The swap and auto-fill APIs require the schedule entry ID. Solution: fetch schedule entries for the selected date via the existing `GET /schedule-entries` endpoint to get the entry ID.

## Files to Create (2)

### `frontend/src/pages/Schedule/components/ScheduleOverview/SwapTeamMemberDialog.tsx`

Swap dialog: load candidates, select replacement, execute swap.

**Props**: `{ entryId: string, teamMemberId: string, teamMemberName: string, onClose: () => void, onSuccess: () => void }`

**State** (object state with typed interface per react-components.md):

```ts
type SwapDialogStage = 'loading' | 'select' | 'swapping'

interface SwapTeamMemberDialogState {
  stage: SwapDialogStage
  candidates: SwapCandidate[]
  swapContext: SwapContext | null
  selectedCandidateId: string | null
  error: string | null
}
```

**Stages**:

1. **Loading** — On mount, calls `listSwapCandidates(entryId, teamMemberId)`. Centered `Loader2` spinner with "Loading replacement candidates..." text. On error: sets stage to `select` with error message. On success: sets stage to `select` with candidates + swapContext.

2. **Select** — Displays:
   - Header: "Swap [memberName]" with ArrowLeftRight icon
   - Subtitle: swap context info (profession + specialty if applicable)
   - Candidate list: each row shows name, profession badge, specialty badge, weekly assignment count (`Badge variant="secondary"` showing "N assignments this week")
   - Selected candidate highlighted with `ring-2 ring-primary-500 bg-primary-50`
   - Error banner (conditional, same pattern as AutoFillDialog)
   - Empty state when no candidates: "No eligible replacements available"
   - Footer: Cancel (secondary) + "Swap" (primary, disabled if no candidate selected)

3. **Swapping** — Calls `swapTeamMember(entryId, { removeTeamMemberId: teamMemberId, addTeamMemberId: selectedCandidateId })`. On success: calls `onSuccess()` then `onClose()`. On error: sets stage back to `select` with error message.

**Error messages** (same parsing pattern as AutoFillDialog):
- `TEAM_MEMBER_NOT_ASSIGNED_TO_ENTRY`: "This team member is not assigned to this schedule entry."
- `TEAM_MEMBER_ALREADY_ASSIGNED_TO_ENTRY`: "This team member is already assigned for this day."
- `PROFESSION_MISMATCH`: "Replacement must have the same profession."
- `SPECIALTY_MISMATCH`: "Replacement must have the same specialty for this slot."

**Modal layout**: Same pattern as AutoFillDialog — fixed inset-0, modal-overlay, card-raised, max-w-xl, max-h-[85vh] overflow-y-auto.

### `frontend/src/pages/Schedule/components/ScheduleOverview/AutoFillDayDialog.tsx`

Auto-fill gaps for a single day: confirm, execute, show result.

**Props**: `{ entryId: string, selectedDate: string, onClose: () => void, onSuccess: () => void }`

**State**:

```ts
type AutoFillDayDialogStage = 'confirm' | 'loading' | 'result'

interface AutoFillDayDialogState {
  stage: AutoFillDayDialogStage
  result: AutoFillDayGapsResponse | null
  error: string | null
}
```

**Stages**:

1. **Confirm** — Header "Auto-fill Day Gaps" with Zap icon. Message: "Automatically assign team members to unfilled slots for {formatDateDisplay(selectedDate)}." Footer: Cancel (secondary) + "Auto-fill" (primary).

2. **Loading** — Centered Loader2 spinner with "Filling gaps..." text.

3. **Result** — Success/warning banner + gap report display:
   - If `!gapReport.hasGaps`: success banner (green, CheckCircle2 icon) "All slots filled! No staffing gaps."
   - If `gapReport.hasGaps`: warning banner (AlertTriangle icon) + profession gap rows (same visual as `GapReportDisplay` but for single-day shape — `professionGaps[]` directly, no `days[]` wrapper)
   - Gap rows: profession label, `assignedCount/requiredCount`, deficit Badge, nested specialty gaps indented with border-l-2
   - Footer: "Close" button → calls `onSuccess()` then `onClose()`

**Error handling**: Same pattern as AutoFillDialog.

**Modal layout**: Same as AutoFillDialog.

## ScheduleOverview Integration

**File**: `frontend/src/pages/Schedule/components/ScheduleOverview/ScheduleOverview.tsx`

Changes:

1. **Add entry ID fetch** — Second `useAPIRequest` to get the schedule entry for the selected date:
   ```ts
   const entriesRequest = useAPIRequest<ScheduleEntryWithAggregateData[]>({
     url: API_ROUTES.scheduleEntries.list,
     params: { startDate: selectedDate, endDate: selectedDate },
   })
   const scheduleEntryId = entriesRequest.state.data?.[0]?.id ?? null
   ```

2. **Add dialog state** — `useDialogs<'swap' | 'auto-fill-day'>()` for dialog management

3. **Add swap target state** — Track which member is being swapped:
   ```ts

     swapTarget: { teamMemberId: string; teamMemberName: string } | null
   }
   ```

4. **Add handlers**:
   - `handleSwap(teamMemberId, teamMemberName)` — sets swap target + activates 'swap' dialog
   - `handleAutoFillDay()` — activates 'auto-fill-day' dialog
   - `handleMutationSuccess()` — calls `apiRequest.actions.refetch()` + `entriesRequest.actions.refetch()`

5. **Pass to AssignedMembersList** — `onSwap={handleSwap}`

6. **Add auto-fill day button** — In the summary card header area (next to "X staff assigned"), add a secondary button with Zap icon "Fill Gaps". Only show when `scheduleEntryId` exists.

7. **Render dialogs conditionally**:
   ```tsx
   {currentActiveDialog === 'swap' && scheduleEntryId && state.swapTarget && (
     <SwapTeamMemberDialog
       entryId={scheduleEntryId}
       teamMemberId={state.swapTarget.teamMemberId}
       teamMemberName={state.swapTarget.teamMemberName}
       onClose={disableDialog}
       onSuccess={handleMutationSuccess}
     />
   )}
   {currentActiveDialog === 'auto-fill-day' && scheduleEntryId && (
     <AutoFillDayDialog
       entryId={scheduleEntryId}
       selectedDate={selectedDate}
       onClose={disableDialog}
       onSuccess={handleMutationSuccess}
     />
   )}
   ```
